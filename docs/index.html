v6 Highlights (2026-04-16)

  • Win32 condvar PI — FUTEX_WAIT_REQUEUE_PI for RtlSleepConditionVariableCS. Kernel atomically requeues waiters from condvar onto CS PI mutex — zero gap. Details →
  • 4th PI coverage path — CS-PI, NTSync PI, pi_cond, and now Win32 condvar PI. Only SRW remains without PI (unsolved in Linux kernel too).
  • condvar-pi max wait: -42% — worst-case tail latency 263→152µs under RT load (500 iterations, 4 load threads)
  • 22/22 RT test suite — condvar-pi test added, no regressions across all existing tests

v5 Highlights (2026-04-15)

  • pi_cond requeue-PI — FUTEX_WAIT_REQUEUE_PI closes the last PI gap across all unix-side condvar consumers (ntdll, audio drivers, gstreamer)
  • SRW lock spin phase — 256-iteration spin before parking, matching Windows behavior. RT threads skip spinning.
  • msvcrt SIMD — AVX memcpy/memmove, SSE2 memchr/strlen/memcmp with runtime CPU dispatch
  • NTSync d8 PI: -52% wait time — deep transitive PI chains now 2x faster
  • rapidmutex: +4.7% throughput, RT max wait 46→36µs
