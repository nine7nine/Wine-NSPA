Architecture & Design

  • Aggregate-Wait and Async Completion CORE
    Aggregate-wait plus same-thread async completion architecture: `NTSYNC_IOC_AGGREGATE_WAIT`, per-process dispatcher-owned `io_uring`, and same-thread CQE drain / reply. Later dispatcher and async-file follow-ons build on this foundation. 4 SVG diagrams.
  • Architecture Overview CORE
    Master system map: client / wineserver / kernel layering, shipped bypass surfaces, residual wineserver floor, and links into the dedicated subsystem docs. 9 SVG diagrams.
  • Audio Stack CORE
    winejack.drv (WASAPI + MIDI on JACK) and nspaASIO bridge: same-period bufferSwitch inside JACK RT callback, futex-synced zero-latency path, unified MIDI handling.
  • Client Scheduler Architecture CORE
    spawn-main + `ntdll_sched` substrate: per-process `wine-sched` / `wine-sched-rt` hosts, async close queue, sched-hosted local timers, and the current client-side helper-thread consolidation model. 3 SVG diagrams.
  • Critical Section PI CORE
    FUTEX_LOCK_PI on CRITICAL_SECTION: fast/slow path, PI chain, and wake / ownership behavior. 3 SVG diagrams.
  • Gamma Channel Dispatcher CORE
    Hybrid ntsync + wineserver request plane: single per-process channel transport, aggregate-wait over channel + uring eventfd + shutdown eventfd, and post-1011 `TRY_RECV2` burst drain after each dispatch.
  • Hook Cache CORE
    Tier 1 + Tier 2 hook caching for SetWindowsHookEx / WH_* dispatch. Drains the per-call get_hook_info round-trip; validated server_dispatch=0 on Ableton 165s session.
  • io_uring I/O Architecture CORE
    Shipped `io_uring` surface: file I/O, dispatcher-owned async `CreateFile`, and socket `RECVMSG` / `SENDMSG`, plus ntsync `uring_fd` integration. 3 SVG diagrams.
  • librtpi (PI mutex / condvar) CORE
    Wine-internal header-only librtpi shim: `pi_mutex_t` / `pi_cond_t` on `FUTEX_LOCK_PI` / `FUTEX_WAIT_REQUEUE_PI`, the NSPA recursive-mutex extension (`NSPA_RTPI_MUTEX_RECURSIVE`), the libclang-based pthread -> pi_* sweep tool, and the `include/rtpi.h` forwarder. Underpins Wine's Unix-side PI coverage.
  • Local-File Bypass Architecture CORE
    Bounded local `NtCreateFile` path for regular files and explicit directories, with shared inode arbitration, selected downstream file fast paths, and lazy server-handle promotion only when the call leaves the local envelope. ~28,500 file opens offloaded per Ableton startup. 2 SVG diagrams.
  • Local Section Bypass CORE
    Client-side unnamed file-backed sections on top of local-file handles: local create / map / query / unmap / close, mapping-bit publication back into the file aggregate, and same-process duplicate promotion at the server boundary. 3 SVG diagrams.
  • Message Ring Architecture CORE
    Same-process cross-thread PostMessage / SendMessage via per-queue memfd rings. Also covers the redraw push ring, the paint-cache fast path, and the MR1 / MR2 / MR4 hardening pass. 5 SVG diagrams.
  • Memory, Sections, Large Pages, and Working-Set Support CORE
    Client-side sections, large-page allocation and mapping semantics, current-process working-set reporting, working-set quota bookkeeping, and the shared-memory backing choices behind request payloads, per-queue memfd regions, shared inode tables, and large-page mappings. 3 SVG diagrams.
  • NT Local Stubs CORE
    Client-side NT bypass catalog: local-file plus local sections, anonymous local events, and sched-hosted `local_timer` / `local_wm_timer`, plus the shared rules for promotion and cross-process arbitration.
  • NTSync PI Kernel CORE
    Wine-NSPA's ntsync kernel overlay: PI primitives, channel transport, aggregate-wait, `CHANNEL_TRY_RECV2`, and the later cache-isolation / lockless-scan hardening work. Production module srcversion `25751C3E41E15401318758E`.
  • NTSync Userspace Sync CORE
    Wine ntdll handle-to-fd cache, server-owned vs. client-created sync-object paths, `linux_wait_objs` / `linux_set_event_obj_pi`, dispatcher channel ioctl wrappers, and PE-side wait coverage. Companion to the NTSync PI Kernel page.
  • Win32 Condvar PI (Requeue-PI) CORE
    FUTEX_WAIT_REQUEUE_PI for RtlSleepConditionVariableCS. Condvar-to-mutex mapping table, 3 new syscalls, zero-gap PI from condvar wake to CS reacquire. 2 SVG diagrams.
  • Wineserver Decomposition PLAN
    Roadmap for splitting wineserver into a thinner residual request service. Aggregate-wait is already shipped; timer/fd-poll/routing decomposition remains ahead.
